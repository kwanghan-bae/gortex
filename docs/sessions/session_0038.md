# Session 0038

## Goal
- 시스템 전역 텍스트 로컬라이징 엔진 구축 (System Localization v1)

## What Was Done
- **docs/i18n/ 디렉토리 및 사전 구축**: 한국어(`ko.json`)와 영어(`en.json`) 기본 사전을 생성하여 시스템 표준 메시지를 키값 기반으로 데이터화.
- **utils/translator.py 수정**: 싱글톤 `SystemTranslator` 클래스와 `i18n` 인스턴스 구현. 런타임에 사전 로드 및 변수 치환(`.format()`) 기능 지원.
- **main.py 연동**: 
    - 부팅 시 `state["ui_language"]`를 기반으로 로컬라이징 엔진 초기화 로직 추가.
    - 하드코딩된 초기화 메시지를 `i18n.t("system.initialized", ...)`로 교체 완료.
- **연속성 확보**: 이제 시스템의 모든 지각적 메시지는 외부 사전을 통해 일관되게 관리되며, 코드 수정 없이 새로운 언어 지원 가능.

## Decisions
- LLM이 직접 생성하는 자유 텍스트(Thought 등)는 원문 보존을 위해 강제 번역하지 않되, 시스템이 내보내는 표준 알림 및 가이드 메시지 위주로 먼저 로컬라이징을 적용함.
- 다국어 사전은 JSON 형식을 사용하여 직관적인 편집이 가능하도록 함.

## Problems / Blockers
- 현재 초기화 메시지 등 일부만 교체된 상태임. 향후 모든 에이전트의 도구 결과 메시지, 오류 알림 등을 전수 조사하여 `i18n.t`로 마이그레이션하는 반복 작업이 필요함.

## Notes for Next Session
- 시스템의 '언어적 일관성'을 위해, 에이전트들이 출력하는 모든 '도구 성공/실패' 메시지들을 `i18n` 키값으로 매핑하고 통합하는 'Message Migration' 작업이 필요함.
