# Next Session

## 세션 목표
- **Self-Patching Semantic Versioning**: `Analyst`가 최근의 진화 강도(수정된 라인 수, 신기술 도입 여부)를 수치화하여, 프로젝트의 버전을 자동으로 패치(0.0.x)하거나 마이너 승격(0.x.0)시키는 자율 버전 관리 루프를 가동한다.
- **Intelligence Density Mapping**: 코드 베이스의 복잡도와 `EvolutionaryMemory`의 규칙 밀도를 결합하여, 시스템의 어느 부분이 가장 지능적으로 성숙했는지 보여주는 '지능 밀도 지도'를 생성한다.

## 컨텍스트
- 시스템이 스스로 기록을 남기기 시작했으므로, 이제는 '버전'이라는 명확한 마일스톤을 스스로 부여할 단계입니다.
- 단순한 파일 구조를 넘어, 지식의 농축도를 파악함으로써 다음 진화 대상을 더 정교하게 선택할 수 있습니다.

## 범위 (Scope)
### 수행할 작업 (Do)
- `VERSION` (신규): 프로젝트의 현재 버전을 저장하는 단일 텍스트 파일 생성.
- `agents/analyst/base.py`: 버전 승격 로직(`bump_version`) 구현.
- `utils/indexer.py`: 코드 라인 수와 규칙 매칭률을 기반으로 한 지능 지수 산출 메서드 추가.

### 수행하지 않을 작업 (Do NOT)
- 사용자 승인 없이 메이저 버전(x.0.0)을 승격시키지 않는다.

## 기대 결과
- 자신의 성장 단계를 버전으로 정의하고 관리하는 '정체성 있는 시스템'으로 진화.
- 지식 밀도 지도를 통해 리팩토링 및 진화의 우선순위를 데이터 기반으로 결정.

## 완료 기준
- `VERSION` 파일이 작업 규모에 따라 자동으로 업데이트되는지 확인.
- 지능 밀도 데이터가 추출되어 로그에 기록되는지 검증.
